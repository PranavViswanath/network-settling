Network Settling Visual Tool

Welcome! This tool lets you input your own grid (like Sudoku or a Latin Square) and see how a network settles on the solution—live and step by step. Change the cell values, press go, and watch as the grid works out the answer via competing activations, not traditional logic. You can experiment to see how different clues shape the solution and how quickly the network "relaxes" into the right state.

What is This?

This is an interactive exploration of connectionism from cognitive science—specifically, what you see in Lecture 7 on Connections. Instead of solving the puzzle through step-by-step elimination, the grid models every possible cell-value assignment as a "unit" with an activation level (how likely that cell is to take on a particular value). Known values ("clues") are set to maximum activation, and the rest start evenly (totally uncertain). As the simulation runs, each cell's possible values compete: if a value is already chosen in the same row or column, its probability drops; if nothing else conflicts, it rises. All these units update together—in parallel—until the solution stabilizes across the whole grid.

Why Make It This Way? The Motivation

I built this as a hands-on project—curious to see connectionist ideas in action, not just theory. I wanted to show that you can set up a problem as a landscape of units and activation, add the right rules as constraints (inhibition for conflicts, strong input for clues), and let the "answer" literally emerge from their interplay—no step-by-step algorithm needed. At first, wrapping my head around the inhibition process took some time: should probabilities for possible assignments drop instantly to zero when there's a conflict, or only when a value's totally ruled out? Working through this logic, tweaking the probability updates, and seeing the network relax over time helped me understand both the computational and cognitive sides of the model.

Why Does It Matter? — Broader CogSci Connections

This approach puts into practice the models discussed in class: it shows how problems can be solved by spreading activation, competition, and constraint satisfaction—not by following internal logic or rules line by line, but by letting the right solution organize itself in a network. This is not just symbolic AI, but connectionism—representing knowledge and solving problems through units (representations), weights (constraints), and iterative stabilization (settling). It's a demonstration of how complex outputs can self-organize from simple local interactions, just like in real neural networks.

Next Steps

This is a first pass—a student-created tool built for a college class in November 2025. Future directions might include visualizing changes in probability more atmospherically, handling larger grids, or even letting users design new constraints (beyond Sudoku-like rules). For anyone interested in how cognitive science theories become functional code, or wanting to literally see parallel constraint satisfaction in action, I hope this inspires you to play and explore further.
